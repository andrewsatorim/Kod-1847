## 3. Cookie-баннер

### 3.1. Требования по закону (152-ФЗ)

- Баннер отображается **при первом визите**, до загрузки любых аналитических скриптов.
- Требует **активного действия** пользователя (нажатие кнопки). Продолжение скроллинга ≠ согласие.
- Срок согласия — **12 месяцев**, затем баннер показывается повторно.
- При отказе или закрытии без действия — аналитические cookie **не загружаются**.

### 3.2. UI-спецификация

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  Мы используем файлы cookie для улучшения работы сайта          │
│  и анализа посещаемости. Подробнее — в Политике cookies.        │
│                                                                 │
│                          [ Принять ]   [ Отклонить ]            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

- **Позиция:** фиксированный блок внизу экрана (`position: fixed; bottom: 0`).
- **Стиль:** фон `#08080A` (или `rgba(8,8,10,0.95)` с backdrop-blur), текст `#F5F0E8`, кнопка «Принять» — `#B89860` (золото), кнопка «Отклонить» — прозрачная с бордером `#9A958B`.
- **Ссылка** «Политике cookies» → `/cookies`.
- **Z-index:** максимальный (выше всех остальных элементов).
- **Анимация:** плавное появление снизу (`transform: translateY → 0`), плавное исчезновение после выбора.

### 3.3. Логика (JavaScript)

```
Файл: /js/cookie-consent.js
```

1. При загрузке страницы проверить `localStorage.getItem('cookie_consent')`.
2. Если значение `accepted` и `cookie_consent_date` < 12 месяцев назад → НЕ показывать баннер, загрузить аналитику.
3. Если значение `declined` и `cookie_consent_date` < 12 месяцев → НЕ показывать баннер, НЕ загружать аналитику.
4. Если нет значения или прошло > 12 месяцев → показать баннер.
5. По нажатию «Принять»:
   - `localStorage.setItem('cookie_consent', 'accepted')`
   - `localStorage.setItem('cookie_consent_date', new Date().toISOString())`
   - Динамически подключить скрипт Яндекс.Метрики.
   - Скрыть баннер с анимацией.
6. По нажатию «Отклонить»:
   - `localStorage.setItem('cookie_consent', 'declined')`
   - `localStorage.setItem('cookie_consent_date', new Date().toISOString())`
   - Скрыть баннер с анимацией.
   - Аналитику НЕ загружать.

### 3.4. Блокировка аналитики

**Критически важно:** скрипт Яндекс.Метрики НЕ должен быть в HTML как обычный `<script>`. Он загружается **только динамически** после получения согласия.

```javascript
function loadYandexMetrika() {
  // Вставить стандартный код Яндекс.Метрики динамически
  (function(m,e,t,r,i,k,a){...})(window, document, "script", "https://mc.yandex.ru/metrika/tag.js", "ym");
  ym(XXXXXXXX, "init", { clickmap:true, trackLinks:true, accurateTrackBounce:true, webvisor:true });
}
```

Номер счётчика Метрики уточнить у заказчика (заглушка `XXXXXXXX`).

## 4. Чекбокс согласия на обработку ПДн

### 4.1. Где размещать

На **каждой форме**, которая собирает персональные данные:

- Форма бронирования стола
- Форма заявки на членство в клубе
- Форма обратной связи
- Любая другая форма с полями имя / телефон / email

### 4.2. UI-спецификация

```
☐ Я даю согласие на обработку персональных данных
   в соответствии с Политикой конфиденциальности
```

- **Чекбокс не предзаполнен** (это требование закона — `checked` по умолчанию запрещён).
- «Политикой конфиденциальности» — кликабельная ссылка на `/privacy`, открывается в новой вкладке (`target="_blank"`).
- Шрифт: Raleway, 13–14px, цвет `#9A958B`.
- Кастомный чекбокс в стилистике бренда (золотая галочка `#B89860` при `checked`).

### 4.3. Валидация

- Кнопка отправки формы **заблокирована** (`disabled`), пока чекбокс не отмечен.
- При попытке отправить без галочки — показать inline-сообщение:  
  «Для отправки заявки необходимо дать согласие на обработку персональных данных»
- Цвет ошибки: `#C75050` (приглушённый красный, не кричащий).

### 4.4. Логика

```javascript
// Для каждой формы
const checkbox = form.querySelector('.consent-checkbox');
const submitBtn = form.querySelector('[type="submit"]');

checkbox.addEventListener('change', () => {
  submitBtn.disabled = !checkbox.checked;
});

form.addEventListener('submit', (e) => {
  if (!checkbox.checked) {
    e.preventDefault();
    // показать сообщение об ошибке
  }
  // Записать факт согласия: дату, время, IP (на сервере)
});
```

## 10. Примечания для Claude Code

1. **consent.html** (Согласие на обработку ПДн) — это справочная страница. Основной механизм согласия — чекбокс на формах. Страница нужна для прозрачности: пользователь может в любой момент прочитать полный текст согласия.

2. **Публичная оферта** `/offer` — самый длинный документ. Если на сайте появится функционал онлайн-оплаты, в форму заказа нужно будет добавить второй чекбокс: «Я ознакомлен и согласен с условиями Публичной оферты».

3. **Порядок подключения скриптов:** сначала `cookie-consent.js`, затем (условно) Яндекс.Метрика. Никакой аналитики в `<head>`.

4. **Фактический адрес клуба** на Арбате — уточнить у заказчика и добавить в колонку контактов в футере отдельно от юридического адреса.

5. **Номер счётчика Яндекс.Метрики** — заглушка `XXXXXXXX`, заменить на реальный.

6. **Email** `info@kod1847.ru` — проверить, что домен и почтовый ящик настроены.
